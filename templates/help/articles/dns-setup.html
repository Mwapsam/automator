DNS records tell the world that mail from your domain is genuinely yours. After provisioning a domain, the domain card on the Domains page lists every record you need — just copy each one into your DNS provider.

The records you'll add

RecordTypeWhat it does
VerificationTXTProves you own the domain so we can switch it on.
DKIMTXTCryptographically signs your mail so receivers can trust it wasn't tampered with.
SPFTXTLists the servers allowed to send for your domain.
DMARCTXTTells receivers what to do with mail that fails the checks above.
Each record has a Name (sometimes called Host) and a Value (sometimes Content or Data). Use the copy buttons on the domain card so nothing gets mistyped.

General steps

  1. Sign in to wherever your domain's DNS is managed (your registrar or DNS host).
  2. Find the DNS / DNS records / Zone editor section.
  3. For each record on the domain card, add a new record with type TXT, paste the Name and Value, and save.
  4. Leave TTL at the default (or "Automatic").

Provider-specific notes

Cloudflare

  • Go to your domain → DNS → Records → Add record.
  • Type TXT, paste Name and Value (Cloudflare calls Value "Content").
  • Set Proxy status to DNS only (grey cloud) — TXT records aren't proxied anyway.

GoDaddy

  • My Products → your domain → DNS → Add.
  • For the Name, if the record is for the root domain GoDaddy uses @. Paste only the host part shown (drop the trailing domain) if GoDaddy appends it automatically.

Namecheap

  • Domain List → Manage → Advanced DNS → Add New Record.
  • Choose TXT Record; the Host field is the Name and Value is the Value.

Google Domains / Squarespace, Route 53, others

  • Look for "DNS", "Custom records" or "Zone file", then add each TXT record with the Name and Value as shown.

Host name tips

Some providers automatically add your domain to the end of the Name. If a record's Name is selector._domainkey.mail.yourcompany.com and your provider already appends mail.yourcompany.com, enter just selector._domainkey. If you're unsure, entering the full name usually still works.

After you add them

  1. DNS changes take time to spread — usually minutes, occasionally a few hours.
  2. Return to Domains and click Check DNS — each record shows whether it's been found.
  3. If it's still pending, just leave the page open; it re-checks automatically every few seconds.
